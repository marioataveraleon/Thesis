\section{Phase 1: Converter Technology benchmark in Equivalent Grid}
The objective of Phase 1 is to perform a comparative analysis of the inverter models available in DIgSILENT PowerFactory, with a particular focus on the differences in dynamic behaviour between grid-following (GFL) and grid-forming (GFM) inverters. This phase is intended to highlight the fundamental response characteristics of both inverter types when the power system is subjected to grid disturbances.

The analysis is conducted by exposing the system to predefined disturbances and observing the resulting frequency and voltage dynamics. By applying identical operating conditions and disturbance scenarios, the comparison isolates the influence of the inverter control philosophy, allowing a clear distinction between GFL and GFM behaviour during transient events.

The power system model used in this phase is the Spanish equivalent grid (Red Eléctrica Española equivalent system), which is widely applied in the context of European grid-code compliance studies. This equivalent grid is defined within the framework of the European Network Codes and their Spanish implementation, and it represents a standardized reference system for the assessment of power generating modules, the grid model is based on the documentation developed by the Working Group for Compliance Monitoring of technical requirements under Regulation (EU) 2016/631 for Power Generating Modules (PGMs). The Technical Standards for the Compliance of Generators (NTS-Generators), version 2.1, published on 9 July 2021, define in Section 7.2 the structure and parameters of the equivalent grid model applicable to the Spanish peninsular electrical system.

\subsection{Grid Model and Test System Definition}

The dynamic system is represented by an equivalent network model based on the Spanish grid code (NTS 2.1), consisting of an equivalent system node (\textit{EQ\_SISTEMA}) and an equivalent power generating module (PGM), denoted as \textit{GEN\_EQ}. The equivalent generator is connected to the system node through a step-up transformer (\textit{TFR\_EQ}), representing the interface between generation and the transmission system.

From the \textit{EQ\_SISTEMA} node, an equivalent transmission line is modelled up to the point of connection (\textit{Nudo PCR}), representing the external grid. The \textit{Nudo PCR} serves as the connection point for the PGM under assessment, including its associated internal network (e.g., evacuation lines and transformers).

The strength of the grid is defined through the short-circuit ratio (SCR), which is fixed to a value of 5 in the base case. This is achieved by setting the short-circuit power at the \textit{Nudo PCR}, denoted as $S_{\mathrm{cc,PCR}}$, accordingly. Based on this value, the equivalent generator base power ($S_{\mathrm{base,EQ}}$) and the transmission system impedance are determined such that the desired grid strength is reproduced.

This formulation ensures that all simulations are performed under consistent and controlled grid conditions, allowing a systematic assessment of the impact of different generation technologies, particularly grid-forming converters, on system stability. \subsection{Simulation Environment}
All simulations are performed in DIgSILENT PowerFactory using the time-domain RMS simulation environment. The system is initialised from a steady-state load-flow solution.
Since RMS simulations are employed, only the positive-sequence representation of the network is considered. The RMS approach is selected as it is sufficient to capture the
electromechanical dynamics relevant for the assessment of system-level stability. To ensure consistency in the power-flow conditions and reactive power control, all
evaluated generating units are configured with the same rated apparent power of 100 MVA, identical step-up transformer parameters, and the same rated power factor of 0.8. Under steady-state conditions, this corresponds to an active power export of 80 MW prior to the application of any disturbance.

The nominal rated power of the built-in PowerFactory models is preserved and scaled to the target capacity using the parallel machines option. This approach ensures that all technologies are assessed under equivalent operating conditions while maintaining
consistent model parameterisation.

\subsection{Disturbances Scenarios}

To evaluate the dynamic behavior of the considered generation technology, two standardized disturbances are simulated under identical operating conditions. These disturbances are selected to represent typical events relevant for power system stability, namely an imbalance between power generation and power consumption, as well
as a severe disturbance affecting the transmission grid.

\subsubsection{Load Step}
A sudden change in active power demand is applied at the network conneection point (PCR Node) to create an positive active power imbalance in the system. The system disturbance occurs at $t = 1\,\mathrm{s}$, the load connected in PCR is of 20 MW, at 15 s the load step occurs of an increase in 25\% rising the active power from 20 to 25 MW. The disturbance is used to asses the frequency response of the system, including frequency deviation and dynamic recovery characteristics. 

\subsubsection{Three-Phase Short Circuit}
A symmetrical three-phase short-circuit is applied at the network connection point (\textit{Nudo PCR}) at $t = 1\,\mathrm{s}$ and cleared after $0.1\,\mathrm{s}$. The selected fault duration is consistent with the recommendations provided in \cite{REE_NTS_2016_631_v21}, which specify a clearing time between 80~ms and 120~ms.
This scenario represents a severe grid disturbance and is used to evaluate the transient voltage stability and frequency response of the system following fault clearance. Furthermore, this disturbance allows the assessment of the low-voltage ride-through
(LVRT) behaviour, as transmission system operators typically require generating units to remain connected to the grid and support voltage recovery during fault conditions \cite{entsoe2016}.


\subsection{Technologies}
The technologies used to achieve a comparative assessment of the dynamic behavior was implemented by first having the behavior of a synchronous machine, the synchronous machine represent the base case and the dynamic behavior of traditional power systems. GFL technology and GFM technology is then introduced and the synchronous machine is substituted either for a GFL or for a GFM inverter to compare the dynamic behaviour. The GFL technology considereed are the GFL Wind Turbine Models for IEC and WECC within PowerFactory, these built in models present the dynamic models of a GFL wind turbine and can be adapted in parameters accordingly to achieve an specific OEM behavior. Moreover, SMA is one of the top invreter providers of RWE, therefore assessing their models to observe the dynamic behavior under disturbances is also relevant and the inverter models for battery and PV are also connected to the system allowing to observe the dynamic behavior. For GFM technology, the built in models of PowerFactory were sibject of study, the Droop Control and the VSM are considered for this study.



The technologies considered for the comparative assessment of dynamic behaviour are introduced sequentially. First, a synchronous generator is used to represent the base case, reflecting the dynamic behaviour of conventional power systems dominated by rotating machines. This reference configuration provides a benchmark against which inverter-based technologies are evaluated.

Subsequently, the synchronous generator is replaced either by a grid-following (GFL) inverter or a grid-forming (GFM) inverter, while all other system conditions are kept unchanged. This approach enables a direct comparison of the dynamic response resulting solely from the different control philosophies.

For the grid-following technology, built-in wind turbine models compliant with IEC and WECC standards available in DIgSILENT PowerFactory are employed. These models represent the dynamic behaviour of GFL wind turbines and allow parameter adaptation to approximate specific original equipment manufacturer (OEM) characteristics. In addition, OEM-specific inverter models provided by SMA for photovoltaic and battery energy storage systems are considered, allowing the assessment of dynamic behaviour based on realistic manufacturer control implementations.

For the grid-forming technology, the built-in PowerFactory models are used. In particular, droop-controlled and virtual synchronous machine (VSM) based grid-forming inverters are considered in this study to evaluate their dynamic behaviour under grid disturbances.

The transmission network operates predominantly at 345 kV. However, the following buses are modeled at different nominal voltage levels in the PowerFactory implementation:

\begin{itemize}

\subsection{Disturbance and Stability Criteria}

A three-phase short-circuit fault is applied at Bus 16 at $t_0 = 1.0$ s and cleared after $100$ ms ($t_c = 1.1$ s). The fault is modeled as a purely reactive impedance with $R_f = 0$ and $X_f = 0.001$~p.u., which corresponds to $X_f = 1.19025~\Omega$ on the system base., consistent across all cases. Bus 16 is selected as a representative disturbance location and corresponds to a load bus within the system

All simulations are performed in RMS with a simulation time of 10 s and an integration step size of $\Delta t = 0.5$~ms. The same numerical settings are maintained for all penetration levels.

Post-fault stability is assessed using:
\begin{itemize}
    \item (i) Voltage recovery at Bus~16,
    \item (ii) Rotor-speed response of the remaining synchronous generators.
\end{itemize}

A case is considered post-fault stable if the bus voltage recovers and remains bounded and if synchronous generator speeds remain synchronized and converge toward a steady value (no growing oscillations or loss of synchronism).

Cases exhibiting sustained solver non-convergence during the post-fault period are classified as numerically not feasible in RMS under the selected disturbance.





\subsection{Generator Replacement Method}


To investigate the impact of increasing converter-based generation, synchronous generators are progressively replaced by grid-following PV model plants while preserving the steady-state operating point of the original IEEE~39 benchmark case. The model used is the OEM SMA grid following PV model. The replacement sequence follows the initial active power dispatch of the synchronous generators, starting with the unit supplying the lowest active power and proceeding in ascending order (Table~\ref{tab:replacement_path}). This dispatch-based criterion ensures a systematic and reproducible replacement path.

Each removed synchronous generator is replaced by a PV plant connected to the same generator bus as the decommissioned unit. The PV plant is rated to match the apparent power of the replaced synchronous generator and is connected through a step-up transformer representing the plant interfacing transformer to the 16.5~kV generator bus.

To preserve the operating point, the PV plant operates under constant voltage control (local controller: Const.~V). The active power setpoint is assigned to reproduce the pre-replacement dispatch of the corresponding synchronous generator. The PV setpoints are scaled such that the aggregated PV output matches the original dispatched P. All remaining synchronous generators retain their original control settings, and Generator G02 remains the slack unit throughout the replacement process.

No modifications are introduced to the network topology, line parameters, transformer data, or load models; only the generation technology at selected buses is altered according to the defined replacement sequence.

The converter penetration level is quantified based on dispatched active power, expressed as a percentage of the total system generation and is calculated as follows, the summary can be seen in the replacement path table \ref{tab:replacement_path}.
\begin{equation}
\mathrm{PV\ penetration}~[\%] = \frac{\sum P_{\mathrm{PV}}}{\sum P_{\mathrm{gen}}}\times 100 .
\end{equation}

\subsection{Small signal stability}
\subsubsection{Small-Signal Stability Analysis Methodology}

Small-signal stability was assessed through eigenvalue analysis of the linearized system model. For each replacement scenario, the system was first solved under steady-state conditions using a power flow calculation. The dynamic model was then linearized around the corresponding operating point and the eigenvalues of the resulting state matrix were computed using the modal analysis tool available in DIgSILENT PowerFactory.

The analysis focuses on the least damped oscillatory modes, i.e., those with the lowest damping ratios or located closest to the imaginary axis. These modes determine how fast the system oscillation decay  govern the dynamic response of the system and are therefore are relevant for assessing small-signal stability.

For each scenario, the eigenvalue spectrum was evaluated and the least damped modes were characterized in terms of their damped frequency and damping ratio. These indicators allow the evolution of system oscillatory behavior to be assessed as synchronous generators are progressively replaced by converter-based generation.

Small-signal stability analysis is performed to evaluate how the dynamic characteristics of the system evolve as the penetration of converter-based generation increases. The replacement of synchronous machines by converter-interfaced resources can alter the dynamic behavior of the power system by modifying the location and damping of the eigenvalues of the linearized system. In particular, converter control dynamics may introduce new modes or shift existing electromechanical modes, potentially affecting system stability. Therefore, eigenvalue analysis provides a useful tool to assess how increasing converter penetration influences the dominant system modes and the overall stability of the operating point.

\section{Phase 3: Network-Constrained Transfer Capability and Hosting Capacity Assessments}

\subsection{Grid Model}

The transfer capability analysis is conducted using the IEEE 39-bus test system described in Phase 2. The same network topology, generation dispatch, and load models are preserved. No replacement of synchronous generators with PV units is considered. 

Only steady-state conditions are considered for this analysis, and all simulations are performed in DIgSILENT PowerFactory.

\subsection{Methodology}

The objective of this phase is to determine the maximum incremental active power injection (hosting capacity) that can be accommodated at each bus of the network without violating operational constraints. Particular emphasis is placed on identifying the limiting network elements (i.e., transmission lines or transformers) that constrain power transfer. This information is highly relevant from a planning perspective, as it enables the identification of grid bottlenecks and supports comparisons with potential network reinforcements, thereby facilitating decisions on suitable locations for new generation or storage projects.

The analysis is based on a sensitivity-driven approach using Power Transfer Distribution Factors (PTDFs), combined with load-flow calculations to accurately determine the loading of network elements under different operating conditions. This approach combines a linear sensitivity-based estimation (PTDF) with AC load-flow evaluation of element loading, thereby improving accuracy while maintaining computational efficiency.

In DIgSILENT PowerFactory, PTDFs are obtained using the \textit{Sensitivities / Distribution Factors} calculation, which linearises the system around an operating point derived from a load-flow solution.

PTDFs quantify the incremental change in active power flow on each network element (in MW/MW) resulting from a 1 MW injection at a given bus, assuming a corresponding withdrawal at the slack bus. In other words, PTDFs indicate how an incremental power injection is distributed across the network elements. It is important to note that PTDF values strongly depend on the network topology and operating point.

For each injection at each bus in the system, PTDF values are computed for all relevant network elements (transmission lines and transformers). PTDF values are also obtained for all contingency topologies. In parallel, the thermal limits of these elements are extracted via Python scripting. For transmission lines, the apparent power limit is calculated from the rated voltage and current as:

\begin{equation}
    S_{\text{max}} = \sqrt{3} \cdot U_{\text{rated}} \cdot I_{\text{rated}}
\end{equation}

For transformers, the rated apparent power is directly used:

\begin{equation}
    S_{\text{max}} = S_{\text{rated}}
\end{equation}

Load-flow calculations are used to obtain the actual power flow of each element for every contingency topology. That is, for each contingency case, a load-flow simulation is performed and the resulting power flows of all network elements are recorded.

Based on the actual power flowing through each element, the available margin is calculated as:

\begin{equation}
    \text{Margin} = S_{\text{max}} - P_{\text{actual}}
\end{equation}

Using the previously computed PTDF values, the maximum allowable additional injection before reaching the limit of each element is estimated as:

\begin{equation}
    P_{\text{max}} = \frac{\text{Margin}}{\text{PTDF}}
\end{equation}

For each bus, the transfer capability is defined as the minimum value of $P_{\text{max}}$ across all network elements. The corresponding element is identified as the limiting element.

All calculations and post-processing steps are automated using Python scripts interfaced with PowerFactory, enabling systematic evaluation of all buses in the test system.

It should be noted that this approach is based on a linear approximation of system behaviour around the operating point. Therefore, its accuracy may decrease for large power injections or in cases where voltage or reactive power constraints become dominant.

\subsection{Contingency Analysis and Security Criteria}

To ensure secure system operation, $N-1$ contingency conditions are explicitly considered in the assessment. Each contingency is simulated by removing a single network element (transmission line or transformer) from service, followed by a load-flow calculation (e.g., using \textit{ComLdf} in PowerFactory

For each contingency scenario, the transfer capability is evaluated using the methodology described in the previous subsection. A transfer is considered feasible only if no network element exceeds its thermal loading limits under both the base case and all contingency conditions.

If a contingency leads to an overload in the base operating point (i.e., without additional injection), the corresponding scenario is considered infeasible for transfer capability assessment, as the system is already operating beyond its secure limits.

The final transfer capability for each bus is therefore defined as the most restrictive value across all considered operating conditions, ensuring compliance with $N-1$ security requirements.
